第一反应当然是：能不能直接给搬瓦工加块硬盘？就像给电脑加内存条那种逻辑，只扩一项，其他配置不动。

直接说结论：**不行。搬瓦工目前不支持单独升级硬盘、内存或 CPU 中的任何一项。** 如果你的磁盘空间不够，只有三条路走：清理腾出空间、整体升级到更高配套餐、或者把数据迁到新机器上。这篇文章把三条路的具体操作都写出来，你对号入座选一个。

## 为什么搬瓦工不支持单独升级硬盘？

这不是搬瓦工的奇怪限制，而是 VPS 行业的通行逻辑。搬瓦工用的是 KVM 虚拟化架构，每台 VPS 的配置（CPU、内存、硬盘、流量）在购买时就和套餐绑定了，底层资源分配是整块的，没有后期单独拆分扩展某一项的设计。

说实话，这个逻辑有它合理的地方。如果你可以随意叠加硬盘、随意叠加内存，套餐体系就乱了——供应商管理难，用户定价也不透明。

换句话说：**想要更大的硬盘，必须换一个硬盘更大的套餐。**

## 第一条路：先把现有硬盘清一清

在决定花钱升级之前，先确认空间是不是真的不够，还是被一堆没用的东西占着。用下来的经验是，很多时候服务器里塞着的日志、临时文件、废弃的安装包，加起来能有好几个 G。

### 查看当前磁盘使用情况

SSH 登进去，两个命令看清楚：

**1. `df -h` — 看整体磁盘占用**

bash
df -h


输出里找 `/` 那一行，`Use%` 就是当前占用百分比。如果超过 85%，确实该处理了。

**2. `du -sh /* 2>/dev/null | sort -rh | head -20` — 找出最占空间的目录**

bash
du -sh /* 2>/dev/null | sort -rh | head -20


这个命令跑完会列出占空间前 20 的目录，照着找就行。常见的大户是 `/var`（日志、数据库）、`/home`（用户文件）、`/tmp`（临时文件）。

### 常见清理操作

**清理系统日志**

bash
# 查看日志占用
du -sh /var/log/

# 清理 30 天前的日志（journalctl 管理的）
journalctl --vacuum-time=30d


**清理软件包缓存（Ubuntu/Debian）**

bash
apt clean
apt autoremove -y


**清理软件包缓存（CentOS）**

bash
yum clean all


**清理 /tmp 目录**

bash
rm -rf /tmp/*


**找出大文件**

bash
find / -type f -size +100M 2>/dev/null


大于 100MB 的文件全列出来，逐个判断要不要删。

---

清理完了再用 `df -h` 看一眼。如果空间够用了，这事儿就解决了，不用花钱。

问题是——如果你的业务本身就在长大，网站访客多了、数据库膨胀了、上传的用户文件越来越多，清理只是缓兵之计，撑不了多久。

## 第二条路：升级到更高配的套餐

这是根本解法。搬瓦工后台支持自助升级套餐，操作很简单，数据不会丢，不需要重装系统。

升级的本质是：**你当前套餐剩余的价值折算成差价，用这个差价补到目标套餐上**。也就是说只需要补缴两个套餐之间的差值，不是重新全额付费。

### 升级套餐的步骤

1. 登录搬瓦工后台（Client Area）
2. 进入「Services → My Services」，找到你的 VPS
3. 点进去，找「Upgrade/Downgrade」选项
4. 页面会列出当前可升级的套餐，以及需要补缴的差价
5. 确认、付款，升级自动完成

整个过程一般 10 分钟内搞定，VPS 会重启一次。数据全部保留。

**注意：升了之后不能直接自助降回来。** 如果升完觉得不合适想降级，要提工单联系客服处理，比较麻烦。所以升之前想清楚目标配置够不够用。

## 第三条路：买新机器做数据迁移

适合这种情况：现有套餐里的其他配置（内存、流量）远超实际需求，只是硬盘不够，而升级到下一档整体费用不划算。

这种情况下，可以考虑单独买一台更大硬盘的新 VPS，用快照功能把数据迁过去。

**搬瓦工的快照迁移流程：**

1. 在 KiwiVM 后台，对旧 VPS 创建快照（Snapshots）
2. 将快照导出
3. 在新 VPS 上导入快照
4. 验证数据完整后，停用旧 VPS

KiwiVM 提供最多 2 个永久快照的免费存储，快照操作在面板里直接点就行，不需要命令行。

## 三条路怎么选

不绕弯，直接给判断建议：

- **空间突然满，但业务量没明显增长** → 先清理，大概率能解决
- **业务在长，数据会越来越多，清理只是延缓** → 直接升级套餐，一步到位，别反复折腾
- **现有套餐其他配置大幅超出需求，只差硬盘** → 考虑买新机迁移，算一下新旧差价再决定

## 关于搬瓦工升级套餐的几个常见问题

**Q：升级套餐后，IP 地址会变吗？**

不会。IP 地址和 SSH 端口保持不变，服务不需要重新配置。

**Q：升级套餐后，能换机房吗？**

能。升级到 CN2 GIA-E 套餐后，可用机房会同步更新到该套餐对应的机房列表，在 KiwiVM 的「Migrate to another DC」里选就行。

**Q：搬瓦工单独升级硬盘这个功能以后会有吗？**

按目前的产品方向来看，基本不会有。搬瓦工的后台升级体系一直是整体套餐升级，没有拆分资源出售的迹象。如果需要弹性资源，可能要看其他支持按需计费的云服务商。

**Q：用 df 命令看到硬盘 100% 了，但业务没跑多少东西，为什么？**

常见原因有两个：一是日志文件堆积（`/var/log` 里的），时间一长能占掉几 G；二是 apt / yum 的包缓存没清理，更新过多次系统的话也会积累不少。先跑一遍清理命令，通常能释放出来一些。

**Q：升级套餐需要停机吗？**

升级过程中 VPS 会重启一次，时间一般在几分钟内。如果有正在运行的服务，建议提前知会相关人员，或者选个流量低谷时间操作。
